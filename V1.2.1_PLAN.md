## Fixes Shipped in v1.2.1

### 🔴 Critical

| # | File | Bug | Fix Applied |
|---|------|-----|-------------|
| C1 | `storage.js:13` | `donut` and `croissant` missing from `DEFAULT_STATE.inventory` — crash on buy for returning players | Added both with value `0` |
| C2 | `main.js` | Stat decay floor inconsistency: load-decay used `Math.max(10)`, live-decay used `Math.max(0)` — stats could silently reach 0 | Both now use `Math.max(5)` for a consistent safety floor |
| C3 | `main.js` | Typing indicator stuck forever if AI API times out or throws — player couldn't send new messages | Added 15-second `setTimeout` safety net that auto-removes the indicator |

### 🟠 High

| # | File | Bug | Fix Applied |
|---|------|-----|-------------|
| H1 | `index.html` + `main.js` | Memory Match race condition: difficulty buttons and game state could mismatch when tapped rapidly | Consolidated all game init into `main.js`; removed duplicate `openMemoryMatch` / `_openMemoryMatch` from `index.html`; added 600ms debounce lock |
| H2 | `main.js` | Daily deal `_computeDailyDeal()` divided by zero when `items.length === 0` | Added early return guard before modulo |
| H3 | `main.js` | `openDressUp()` called `state.pet.species` without null check — crash if called before pet creation | Added `if (!state.pet?.species) return;` guard |

### 🟡 Medium

| # | File | Bug | Fix Applied |
|---|------|-----|-------------|
| M1 | `main.js` | `todayKey()` returned UTC date — daily deals and streaks reset at UTC midnight, not player's local midnight | Switched to local date using `new Date()` fields |
| M2 | `main.js` | Memory Match `onEnd` callback did `awardXP` before `saveGame` — XP lost if browser closed between the two | `saveGame(state)` now called immediately after `awardXP`, before any UI update |
| M3 | `main.js` | Chat history trimmed at 40 messages but no localStorage size cap | Trimmed to 30 messages |
| M4 | `main.js` | Memory Match result panel not cleared when switching difficulty — old result could show over new game | Added `el('mm-result-overlay').classList.remove('open')` at start of `openMemoryMatch` |

### 🟢 UX Improvements

| # | What Changed | Where |
|---|--------------|-------|
| U1 | Feed modal now shows "Need X more 🪙" for items within 50% of afford price | `openFeedModal()` in `main.js` |
| U2 | Chat input auto-focuses after pet responds — no need to tap input again | `handleChatSend()` in `main.js` |
| U3 | 375px phone media queries — tighter padding, smaller fonts on smallest screens | `css/style.css` |
| U4 | Locked food items shown with orange hint text instead of disappearing silently | `food-btn-locked` CSS class |

## Deferred to v1.3

These were identified in the QA audit but are **enhancements, not blockers**:

- Confetti canvas resize on phone rotation
- First-time tutorial / tip overlay
- "Pet unresponsive" state when all stats < 10% for 3+ days
- Shop inventory cap (max 20 per item type)
- Background music loop with per-screen fade
- localStorage size warning when approaching quota
